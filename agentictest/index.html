AGENTIC DESIGN PATTERNS

Fourteen agentic AI patterns from the book, each reimagined as an interactive visual music application built with the Web Audio API.

Based on "Agentic Design Patterns: A Hands-On Guide to Building Intelligent Systems" by Antonio Gulli

Part One — Core Patterns

Foundational patterns for building agentic systems: chaining, routing, parallelization, reflection, tool use, planning, and multi-agent coordination.

Ch1

Tone Chain

Prompt Chaining Pattern

Sound flows through a sequential pipeline of transformation stages. Each node processes and forwards to the next — source, pitch shift, filter, delay, reverb — producing an audibly evolving signal from input to output.

→
Ch2

Sound Router

Routing Pattern

Play notes via keyboard or mouse. Each note is classified by frequency and dynamically routed to one of four specialized sonic pathways — Crystal, Warm, Pulse, or Deep — with animated particles showing the routing decision in real time.

→
Ch3

Parallel Synth

Parallelization Pattern

Four independent sound agents — Drums, Bass, Melody, and Ambient — generate musical layers concurrently. Each runs its own sequencer, and all outputs merge through a shared audio graph into a unified mix.

→
Ch4

Evolving Melody

Reflection Pattern

Random melodies are generated and scored by a critic on consonance, contour, and rhythm variety. The best melody survives and mutates into the next generation. Watch fitness climb as the reflection loop iteratively refines the music.

→
Ch5

Sound Toolkit

Tool Use Pattern

Describe a mood and an agent selects the right instruments from its toolkit — synth pads, drum patterns, bass lines, arpeggios, reverb, and filters. Watch it think, choose, invoke each tool, and build a soundscape from scratch.

→
Ch6

Song Architect

Planning Pattern

Pick a genre, mood, and tempo. The system decomposes the goal into a structured plan — Intro, Verse, Chorus, Bridge, Outro — then executes each section in sequence, building up instrumentation as the song progresses.

→
Ch7

Jazz Ensemble

Multi-Agent Pattern

Four AI musicians — Piano, Bass, Drums, and Sax — improvise together. They communicate via visible messages, rotate leadership, and adapt to each other's playing. Emergent music through agent collaboration.

→

Part Four — Advanced Patterns

Production-grade patterns for communication protocols, resource management, reasoning, safety, monitoring, prioritization, and autonomous exploration.

Ch15

Network Symphony

Inter-Agent Communication (A2A) Pattern

Eight musical agents across different "frameworks" discover each other, exchange Agent Cards, negotiate protocols, and form ensembles. Watch the network self-organize through discovery, negotiation, and collaborative performance phases — with every A2A message visible in a protocol inspector.

→
Ch16

Budget Orchestra

Resource-Aware Optimization Pattern

A 16-step sequencer with a depleting compute budget. Each note costs credits based on its quality tier — Premium, Standard, or Economy. An auto-optimizer allocates resources intelligently, and fallback mode gracefully degrades when the budget runs low. Full analytics dashboard tracks cost, quality, and efficiency.

→
Ch17

Composition Reasoner

Reasoning Techniques Pattern

Watch a composition built through explicit reasoning. Chain-of-Thought plans the melody step by step, Tree-of-Thought explores branching chord progressions with scored alternatives, and a ReAct loop observes, thinks, acts, and self-corrects. Click tree nodes to audition chords, then play the full composed piece.

→
Ch18

Guarded Synth

Guardrails / Safety Pattern

An interactive synthesizer where every action passes through 5 visible guardrail gates — input validation, rate limiting, harmony checking, output filtering, and safety monitoring. Toggle gates on and off, run stress tests, and watch the pipeline catch unsafe audio in real time with animated pass/block indicators.

→
Ch19

Performance Lab

Evaluation and Monitoring Pattern

A professional monitoring dashboard tracking three music generator agents. Live metric charts, trajectory analysis comparing expected vs actual output, A/B comparison with voting, drift detection with alerts, and anomaly injection. Sparklines, severity-tagged logs, and adjustable thresholds throughout.

→
Ch20

Priority Mixer

Prioritization Pattern

Sound requests compete for 6 limited audio channels. A priority engine scores each by urgency, importance, dependencies, and freshness. Watch the queue sort in real time, see higher-priority sounds preempt lower ones, and stress-test with a flood of requests. 14 synthesized sound types with full scheduling logic.

→
Ch21

Sound Explorer

Exploration and Discovery Pattern

Navigate a fog-of-war 24x24 musical landscape with four distinct regions — Melodic Plains, Rhythmic Caves, Harmonic Forest, and Experimental Void. Discover unique sounds, mark rare finds, build a collection, and combine discoveries. Auto-explore mode with pathfinding seeks out the unknown.

→
Each app is a single HTML file using the Web Audio API — no external dependencies. Click any card to launch.
